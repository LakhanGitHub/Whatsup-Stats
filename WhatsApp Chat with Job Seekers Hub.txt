powershell

# Define the list of servers
$servers = "Server1", "Server2", "Server3"  # Replace with your server names

# Define log file
$logFile = "C:\PatchLogs\PatchResults_$(Get-Date -Format 'yyyyMMdd_HHmmss').txt"

# Create log directory if it doesn't exist
New-Item -ItemType Directory -Path "C:\PatchLogs" -Force | Out-Null

# Function to log messages
function Write-Log {
    param($Message)
    $timestamp = Get-Date -Format "yyyy-MM-dd HH:mm:ss"
    "$timestamp - $Message" | Out-File -FilePath $logFile -Append
}

# Loop through each server
foreach ($server in $servers) {
    Write-Log "Starting patch process for $server"
    try {

3. Handling Failed Patches
If patches fail, you can extend the script to detect failures and take corrective actions:

powershell

# Enhanced script with failure handling
foreach ($server in $servers) {
    Write-Log "Starting patch process for $server"
    try {
        if (Test-Connection -ComputerName $server -Count 2 -Quiet) {
            Write-Log "$server is online. Proceeding with updates."

                # Check if updates were installed successfully
                $failedUpdates = $updates | Where-Object { $_.Result -ne "Installed" }

                    Updates = $updates
                    FailedUpdates = $failedUpdates

}

PSWindowsUpdate Module: Manages Windows Update operations via PowerShell.

Retry Logic: Attempts to reinstall failed updates.
Logging: Tracks success, failures, and errors in a log file.

Clear the update cache: Remove-Item -Path "C:\Windows\SoftwareDistribution\Download\*" -Recurse -Force.
Ensure enough disk space is available.
Manual Intervention: For persistent failures, connect to the server and run updates manually via GUI or CLI to diagnose further.
